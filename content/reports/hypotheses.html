Region 7:

Using the best model (with the triple interaction), let’s test two hypotheses:

  1. One-sided test: the triple interaction between subject control, ungrammatical sentences and mismatches is greater than 0 (Q1)
  2. Two-sided test: the triple interaction is equal to 0 (Q2)
Code 
m <- brm(file = paste0(relative_path, 'models/r7/fp-m7'))

q1_formula <- 'controlsubject : grammaticalungrammatical : distractormismatch > 0'
q2_formula <- 'controlsubject : grammaticalungrammatical : distractormismatch = 0'

hyp <- hypothesis(m, c(q1 = q1_formula,
                       q2 = q2_formula)
                       )
print(hyp)
Hypothesis Tests for class b:
  Hypothesis Estimate Est.Error CI.Lower CI.Upper Evid.Ratio Post.Prob Star
1         q1      0.1      0.04     0.03     0.17      86.72      0.99    *
2         q2      0.1      0.04     0.01     0.18         NA        NA    *
---
'CI': 90%-CI for one-sided and 95%-CI for two-sided hypotheses.
'*': For one-sided hypotheses, the posterior probability exceeds 95%;
for two-sided hypotheses, the value tested against lies outside the 95%-CI.
Posterior probabilities of point hypotheses assume equal prior probabilities.

A lot of this is customizable but this is already quite useful. The first few columns tells us the effect of what we are testing (here: controlsubject : grammaticalungrammatical), with a mean estimate effect of 0.1 (Estimate), an estimated error of 0.04 (Est. Error), and the lower and higher credible intervals for the value. Note that the values are slightly different for Q1 and Q2 even though they are testing the same effect. The reason is that we are sampling from the model’s posterior, so there’s some small variation if we don’t set the same seed for both hypothesis tests.

For one-sided tests we also get an evidence ratio (Evid.Ratio). This is the evidence in favor of the hypothesis (here: effect > 0) divided by its complement (here: effect < 0), based on the posterior probability of the effect. 86.72 means that there’s very strong evidence in favor of the effect being greater than 0. You also get the posterior probability (Post.Prob) for the hypothesis you’re testing: how likely it is that the value is greater than 0. These two quantities are the same. Evid. Ratio is just Post.Prob / (1 - Post.Prob) but some people find one more intuitive than the other. I think Post.Prob is easier to grasp.

The evidence ration for a two-sided test requires us to re-compute a few things so I am skipping over it and it just shows NA but we should talk about this when we have a clear set of hypotheses.

Finally, the star * indicates whether the posterior probability either lies above/below the value for more than 95% of its values (one-sided), or whether the value tested against lies outside the 95% CI (two-sided).

Summary

You can ask a lot more interesting questions about the way the parameters interact. Here I was just doing stuff like distractormismatch > 0 but nothing is stopping you from doing distractormismatch + distractormismatch:grammaticalungrammatical > 0 or similar. It all depends on what we want to know. I had never used this functionality but it’s quite convenient. So let’s maybe sit down and think about concrete things one may want to ask the models. I can also share the models with you if that’s easier.
